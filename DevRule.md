AI 开发协作规则

=== 工作流程 ===

1. 代码修改后的标准流程：

   - 运行 linter 检查警告
   - 运行测试确保通过
   - 使用格式化工具格式化代码

2. 警告处理原则：
   - 未使用的变量/参数加前缀 `_` 避免警告
   - 移除未使用的导入
   - 替换过时的 API

=== 代码风格禁止事项 ===

1. 禁止在代码和文档中使用 emoji
2. 禁止在代码中保留调试语句（println/print）
3. 禁止保留注释掉的旧代码
4. 禁止保留无意义的 TODO 注释

=== 性能优化要求 ===

⚠️ 禁止未经测试就声称"更快"、"优化"、"高性能"

- 任何性能相关声明必须有基准测试数据支持
- 格式：优化前 XXms → 优化后 YYms（提升 ZZ%）

=== Git 提交协助 ===

当用户要求上传到 git 时：

1. 写一份简短的英文 commit 信息（遵循 Conventional Commits）
2. 创建版本文档：V 版本号\_改动标题.md（例如 V1.2.0_AddUserAuthentication.md）
3. Commit 格式：feat/fix/docs/refactor/test/chore

=== 项目特定规则 ===

1. 测试文件命名：\*\_test.mbt
2. 构建产物目录：target/
3. 临时文件不提交到版本控制
